Example of a simple reports

Here's case, where there are "bad" callouts from the Ivy file - we're calling everything out explicitly:

       <dependency org="org.apache"           name="commons-dbcp"  rev="1.2.1"          conf="build" />
       <dependency org="org.apache"           name="commons-collections"   rev="3.2"    conf="build" />
       <dependency org="org.apache"           name="commons-pool"  rev="1.0.1"          conf="build" />
       <dependency org="org.jdom"             name="jdom"          rev="1.0"            conf="build" />

Notice that we now have some red lines - we're calling out commons-pool 1.0.1 when commons-dbcp is requesting 1.3, and we're also requesting commons-collections 3.2 when commons-dbcp is also asking for revision 3.2. In both cases, we can remove the explicit callout from our ivy file, and clean things up a bit:

       <dependency org="org.apache"           name="commons-dbcp"  rev="1.2.1"          conf="build" />
       <dependency org="org.jdom"             name="jdom"          rev="1.0"            conf="build" />

Now we see we get the same results, but with less clutter in the Ivy file - we only need to call out two dependencies instead of four!
